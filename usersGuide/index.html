Downloads and Installation

netCDF

Getting your files into netCDF format will require downloading the netCDF software. We recommend building with HDF5. This gives support for netCDF4 and allows the netCDF file to include groups of data. So first, you will need to get HFD5.You will also need the zlib distribution. Unidata offers a complete set of installation instructions here.

Python

Fast version: Use the Enthought python distribution, which includes many useful scientific libraries, including numpy and netCDF4 which will be needed for this project.

Longer version: install python, numpy, and netCDF4 yourself. Python 2.7 is recommended. For interfacing with the netCDF software, we are using the excellent python module netCDF. You will need to first install the python scientific library numpy.

campbellNetCDF Code

All of our code is posted on github. You can download a zip file of the current version here.

Making the metadata XMLs

Program xmls

The observation xml has to be created based on manual inputs, but there are several ways to help automate the process. We have a command-line utility which scrapes every variable name from a set of program xmls, and prompts the user to supply an observation name (if exists) for each variable. This can be tedious (since many observation names are shared across variables), so we also allow for the building of the xml from a simple user-created csv file. Lastly, we also offer a utility which will guess at variable names and groups--guesses which will mostly be useful only if you closely follow the Campbell naming conventions set up in sample code in program manuals :).
