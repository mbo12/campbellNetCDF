Campbell to NetCDF Utilities

We use Campbell Scientific dataloggers in our environmental research, and like many users, we have multiple loggers at just one site, each collecting multiple data tables. For ease of analysis, we want to herd these tables into a single file or database that supports querying across all of our information. Further, we need to document metadata from a diverse set of sources; this extra documentation is key for ensuring that the data can be shared across labs.

Here's our solution: a set of utilities for processing the Campbell Scientific data file format into a self-described netCDF file. Along the way, we provide support for generating xml files of additional metadata such as calibration schedules, instrument information, and observation units.

Our work flow is to make a single daily netcdf file for each day in the record and to then create version of the file as we engage in post-processing. However, we aim to be flexible by creating some configuration files so you can target these utilities to your needs. Or, the complete source is available on github, so feel free to fork our project!
