Why NetCDF?

When looking for a data storage solution, we wanted a system that would make it easy for us to analyze the data and for our collaboraters to work with it as well. The NetCDF file format has allowed us to accomplish both of these goals.

  • Attributes We love the ability to include all relevant metadata for a variable: units, processing instruction, instrument measured by, observation of, location collected at, etc.
  • Uniform Structure Whenever we get a new instrument for our field site, our data table schema has to change. Analyzing these csv files becomes a pain: the processing script needs to handle a number of different structures. When processing via NetCDF, we simply have a uniform structure for the whole tower record. Values are filled with NaNs for periods when a sensor was not being recorded. A consistent structure allows for a single processing script, which saves a researcher's time and simplifies code.
  • Groups NetCDF4 is built on top of HDF5. This means it supports a hierarchial grouping of data. We use this feature to group by epoch (the turbulent 10Hz data is in a separate group). Grouping by sampling frequency has made our post-processing easier.

Why XML?

XML has given us the flexibility to define a wide-range of metadata while maintaining some structure.
